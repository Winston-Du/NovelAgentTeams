# 如何调整剧情走向与埋设暗线/支线

> 适用于：novels_project CrewAI 小说创作系统
> 最后更新：2026-03-07

---

## 总览

本系统的创作流水线是 **总编驱动** 的：

```
总编（大纲）→ 人物设计师（状态卡）→ 剧情撰写员（正文）→ 资深校对（优化+摘要）
```

要调整剧情或埋暗线，你需要在合适的环节注入你的意图。下面按**具体操作步骤**展开。

---

## 第一步：编写你的剧情调整指令

在开始修改任何文件之前，先把你的意图写清楚。建议创建一个文件来管理：

### 1.1 创建剧情调整文件

在 `DESIGN/` 目录下新建文件 `DESIGN/plot_directions.yaml`：

```yaml
# 剧情调整指令 - 每次运行前按需修改此文件

# ========== 整体剧情调整 ==========
plot_adjustments:
  - chapter_range: "第12章-第15章"
    description: "加速推进关税争夺，提前引入官府势力"
    reason: "前面铺垫过长，需要加快节奏"

  - chapter_range: "第16章起"
    description: "陆商曜开始暗中布局，表面示弱实则蓄力"
    reason: "为卷一高潮做准备"

# ========== 暗线（长期伏笔，跨多章回收）==========
hidden_plotlines:
  - id: "dark_line_001"
    name: "契约古印的副作用"
    plant_chapter: 12        # 从哪章开始埋
    recover_chapter: 20      # 预计哪章回收
    description: "每次使用古印后，陆商曜手背会出现淡淡的黑纹，但他自己没注意到"
    planting_method: "通过木九公偶然瞥见手背的描写暗示，不要点明"
    recovery_method: "在关键契约对决中古印失控，黑纹蔓延"

  - id: "dark_line_002"
    name: "周桓背后的势力"
    plant_chapter: 13
    recover_chapter: 18
    description: "黑商周桓并非单独行动，背后有人指使"
    planting_method: "周桓偶尔说出超出他智力的话，或提到'上面的意思'"
    recovery_method: "幕后人物正式登场"

# ========== 支线（独立小故事线）==========
subplots:
  - id: "subplot_001"
    name: "铁阙的身世"
    active_chapters: [14, 16, 19]
    description: "铁阙在某些场合表现出异常的警觉，暗示他有军旅背景"
    resolution: "第19章揭示铁阙是逃兵，为后续招募做铺垫"

  - id: "subplot_002"
    name: "市集暗流"
    active_chapters: [13, 15, 17]
    description: "市集上多个小商贩暗中联络，形成反周桓联盟"
    resolution: "第17章集体站出来支持陆商曜"
```

### 1.2 你需要想清楚的事

写每一条暗线/支线时，回答四个问题：

| 问题 | 为什么重要 |
|------|-----------|
| **在哪里埋？** | 总编需要知道从哪章开始布局 |
| **怎么埋？** | 撰写员需要知道用什么手法（对话暗示/环境细节/行为异常） |
| **在哪里收？** | 总编需要规划回收节奏 |
| **怎么收？** | 撰写员需要知道揭示方式 |

## 第二步：修改总编 Prompt — 让大纲纳入你的设计

文件路径：`DESIGN/PROMPTS/chief_editor_prompt.md`

### 2.1 在「任务」部分添加剧情指令读取

找到 `## 任务` 部分（约第19行），在原文后追加：

```markdown
## 剧情调整指令（必须遵守）

在制定章大纲时，你必须参考 `plot_directions.yaml` 中的以下内容：

### 暗线埋设规则
- 检查当前章节是否在某条暗线的 `plant_chapter` 范围内
- 如果是，必须在大纲的 `foreshadowing` 中安排对应的暗线埋设
- 暗线的埋设方式必须按照 `planting_method` 执行，**禁止直接点明**
- 检查当前章节是否在某条暗线的 `recover_chapter`，如果是，安排伏笔回收

### 支线安排规则
- 检查当前章节是否在某条支线的 `active_chapters` 中
- 如果是，在大纲的故事结构中安排支线推进
- 支线不能喧宾夺主，篇幅控制在全章的 20% 以内

### 剧情调整规则
- 检查当前章节是否在某条调整的 `chapter_range` 内
- 如果是，大纲的整体方向必须服从调整指令
```

### 2.2 扩展输出格式

找到 `### 5. 伏笔埋设（foreshadowing）` 部分（约第57行），改为：

```markdown
### 5. 伏笔与暗线（foreshadowing）
- content：要埋设的内容
- type：「伏笔」/「暗线」/「支线推进」
- source：来自 plot_directions.yaml 的哪条指令（如 dark_line_001），如果是原创伏笔则写「原创」
- planting_method：具体的埋设手法
- recovery_plan：预计回收位置和方式
```

## 第三步：修改 inputs 传入你的剧情指令

文件路径：`run.py`

### 3.1 在 `load_chapter_outline()` 中加载剧情指令

在 `run.py` 的 `main()` 函数中，找到准备输入数据的部分（约第251行 `inputs = {`），改为在构建 inputs 前加载你的指令文件：

```python
# 加载剧情调整指令
plot_directions_file = Path("DESIGN/plot_directions.yaml")
plot_directions = {}
if plot_directions_file.exists():
    with open(plot_directions_file, 'r', encoding='utf-8') as f:
        plot_directions = yaml.safe_load(f) or {}
    print(f"📋 已加载剧情调整指令")
    # 筛选当前章节相关的指令
    chapter_id = args.chapter
    active_dark_lines = [
        dl for dl in plot_directions.get('hidden_plotlines', [])
        if dl.get('plant_chapter', 999) <= chapter_id <= dl.get('recover_chapter', 0)
    ]
    active_subplots = [
        sp for sp in plot_directions.get('subplots', [])
        if chapter_id in sp.get('active_chapters', [])
    ]
    active_adjustments = [
        adj for adj in plot_directions.get('plot_adjustments', [])
    ]
    print(f"   活跃暗线: {len(active_dark_lines)} 条")
    print(f"   活跃支线: {len(active_subplots)} 条")
else:
    active_dark_lines = []
    active_subplots = []
    active_adjustments = []
    print("📋 未找到剧情调整指令文件，使用默认流程")

# 准备输入数据
inputs = {
    **chapter_info,
    "character_base_cards": character_cards,
    "plot_directions": plot_directions,
    "active_dark_lines": active_dark_lines,
    "active_subplots": active_subplots,
    "active_adjustments": active_adjustments,
}
```

这样四个 Agent 都能在 inputs 中读到你的剧情意图。

## 第四步：修改撰写员 Prompt — 让正文执行你的暗线

文件路径：`DESIGN/PROMPTS/plot_writer_prompt.md`

### 4.1 在「核心创作原则」后添加暗线写作指南

找到 `## 核心创作原则` 部分（约第130行），在其后追加：

```markdown
## 暗线与支线写作指南

### 暗线写作原则（最重要！）
暗线的核心是 **读者第一遍读不出来，第二遍恍然大悟**。

**埋设手法**：
1. **环境暗示**：通过场景细节暗示（如"他手背上似乎多了一道浅纹"）
2. **对话夹带**：在正常对话中夹带一句异常的话，但不做解释
3. **行为异常**：人物做出与性格微微偏离的举动，但叙述上一笔带过
4. **视角遮蔽**：通过主角视角的局限性自然地隐藏信息

**禁止做法**：
- ❌ 不要用"他不知道的是..."这种上帝视角暴力暗示
- ❌ 不要让人物突然说出暗线相关的关键词
- ❌ 不要用内心独白解释暗线
- ❌ 不要为了埋暗线打断正常叙事节奏

**检查标准**：暗线内容删掉后，章节读起来应该完全通顺。暗线是"额外的层次"，不是"必要的情节"。

### 支线写作原则
- 支线篇幅不超过全章 20%
- 支线必须与主线有情感或主题上的呼应
- 支线人物的出场要自然，不能"为了支线而出场"
```

文件路径：`DESIGN/PROMPTS/proofreader_prompt.md`

### 5.1 在校对检查项中添加暗线审查

找到 `### 校对优先级` 部分（约第24行），在最后追加一项：

```markdown
5. **暗线审查**：检查暗线埋设是否自然、是否过于明显、是否打断叙事
```

### 5.2 在自检清单中添加暗线检查

在校对 Prompt 的自检清单部分追加：

```markdown
### F. 暗线与支线检查
- [ ] 暗线埋设是否足够隐蔽？（删掉暗线内容后章节是否通顺）
- [ ] 暗线是否按照 plot_directions.yaml 中的 planting_method 执行？
- [ ] 支线篇幅是否控制在 20% 以内？
- [ ] 支线是否与主线有呼应？
- [ ] 是否有应该在本章回收的伏笔/暗线被遗漏？
```

## 第六步：更新人物卡库（如需要）

文件路径：`src/novels_project/config/character_base_cards.yaml`

如果你的剧情调整涉及角色变化，需要同步更新人物卡。

### 常见场景

| 剧情调整 | 人物卡改动 |
|---------|-----------|
| 新增重要配角 | 在 `a_tier` 或 `s_tier` 下添加完整人物卡 |
| 角色性格变化（黑化/成长） | 更新 `core_personality`、`character_flaw`、`unique_speaking_style` |
| 角色关系变化 | 更新相关角色的 `core_motivation` |
| 暗线涉及角色隐藏身份 | **不要**写进人物卡（否则 Agent 会直接暴露），在 `plot_directions.yaml` 中管理 |

**重要**：角色的隐藏信息（如铁阙是逃兵）不要写进 `character_base_cards.yaml`，因为所有 Agent 都能读到这个文件。隐藏信息只放在 `plot_directions.yaml` 中，仅通过总编大纲间接控制。

## 第七步：运行并验证

### 7.1 先用 dry-run 检查输入是否正确

```bash
python run.py --chapter 12 --dry-run
```

确认日志中显示了你的暗线/支线被正确加载。

### 7.2 运行单章测试

```bash
# 标准模式
python run.py --chapter 12

# 迭代模式（推荐，校对会额外检查暗线质量）
python run.py --chapter 12 --iterative --quality-threshold 7
```

### 7.3 检查输出

查看 `output/raw_outputs/chapter_12_raw.yaml`，确认：

1. **总编大纲**的 `foreshadowing` 部分是否包含了你的暗线
2. **章节正文**中暗线埋设是否自然
3. **校对报告**中是否有暗线相关的审查意见

## 快速参考：改动清单

每次调整剧情时，按以下清单检查：

```
□ 1. 编写/更新 DESIGN/plot_directions.yaml（你的剧情意图）
□ 2. 确认 chief_editor_prompt.md 包含暗线读取指令（首次需要改，之后不用再改）
□ 3. 确认 plot_writer_prompt.md 包含暗线写作指南（首次需要改，之后不用再改）
□ 4. 确认 proofreader_prompt.md 包含暗线检查项（首次需要改，之后不用再改）
□ 5. 确认 run.py 加载了 plot_directions.yaml（首次需要改，之后不用再改）
□ 6. 如有角色变化，更新 character_base_cards.yaml
□ 7. dry-run 验证输入正确
□ 8. 运行并检查输出
```

**首次配置完成后，日常只需要改第 1 步和第 6 步**（更新 `plot_directions.yaml` 和人物卡），其余文件不用再动。
