## 🎯 人物管理规则

### 主线人物（character_type: main）
```yaml
陆商曜:
  character_type: "main"  # 完整追踪
```
- 📍 位置：base_cards中（S_TIER）
- 📌 处理：每章生成详细状态卡
- 📈 追踪：跨章追踪状态变化
- 💾 存储：章节摘要中持续维护

**特征**：
- 贯穿全小说的关键角色
- 需要详细的性格、动机、行为预测
- 每章都要反映其状态变化

---

### 支线人物（character_type: supporting）
```yaml
铁阙:
  character_type: "supporting"  # 需要时追踪
```
- 📍 位置：base_cards中（A/B/C_TIER）
- 📌 处理：本章出现时生成中等详度状态卡
- 📈 追踪：仅在出现章节追踪
- 💾 存储：章节摘要中保存

**特征**：
- 多次出现但非核心角色
- 需要在多章中保持连贯性
- 出现时要体现其性格特点

---

### 临时人物（仅在outline指定）
```markdown
**临时人物**（仅本章）：
- 赤眉匪头（临时反派）
  - 身份：盗匪头目
  - 动机：抢劫试探
  - 说话风格：粗野蛮横
```
- 📍 位置：章节outline中指定
- 📌 处理：根据outline临时创建简化卡
- 📈 追踪：无（仅本章）
- 💾 存储：本章使用后丢弃

**特征**：
- 一次性出场或极少重复
- 无跨章连贯性需求
- 角色描述在outline中简化指定

## 🔄 使用流程

### 创建新章节时

```
Step 1: 编写 outline.md
        ├── 指定本章主故事线
        ├── 列出涉及的base_cards人物
        │   └── 只写涉及的，不写全部
        └── 定义本章新增临时人物
            ├── 临时反派
            ├── 临时配角
            └── 背景人物

Step 2: 人物设计师Agent处理
        ├── 读取base_cards
        ├── 对每个人物检查character_type
        ├── 确定处理方式：
        │   ├── type=main → 生成详细状态卡
        │   ├── type=supporting且本章涉及 → 生成中等状态卡
        │   ├── type=supporting且本章不涉及 → 跳过
        │   └── 仅在outline中 → 根据描述临时生成简化卡
        └── 输出 chapter_character_states.yaml

Step 3: 剧情撰写员使用状态卡进行创作

Step 4: 完成后存档到 chapter_summaries/
```

## 💡 最佳实践

### 何时添加到base_cards

✅ **添加为支线人物**：
- 将在2个或更多章节出现
- 需要跨章连贯性
- 与主线有重要互动

❌ **改为临时人物**（仅在outline指定）：
- 只在1-2个场景出现
- 一次性角色
- 无跨章承接需求

### 支线人物的详度平衡

```
base_cards中的描述详度：
├── 主线人物：完整的背景、性格、复杂的行为预测
├── 支线人物：核心特征 + 关键动机 + 基本说话风格
└── 临时人物：role + 2-3个性格关键词 + 动机
```

### 临时人物在outline中的指定示例

```markdown
### 本章临时人物

**赤眉匪头（本章关键临时角色）**
- 身份：盗匪头目，率众袭击货栈
- 核心性格：粗野、蛮横、缺乏理性
- 动机：抢劫财货，对陆商曜试探底线
- 说话风格：方言、威胁、粗暴词汇多
- 出场情景：第三幕，组织50人左右的匪徒攻击
- 与陆商曜关系：陌生，首次碰撞

**市集散户 × 3（临时配角）**
- 身份：普通商人，看热闹的
- 性格关键词：精明、议论纷纷
- 说话风格：方言、议论、评头论足
- 出场情景：第二/三幕，作为背景人物，评论事件发展
- 作用：衬托、反映舆论
```
